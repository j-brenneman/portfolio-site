BookShelf

BookShelf

Description

This app allows a user to search books by title and author. It utilizes the Google Books API and jquery autocomplete to make the search effective an UI/UX friendly. Once the user finds a book of interest they can add quotes and save it to their profile. They can then pull up all their favorite books and quotes.

This app was a personal project and I worked hard to make the user experience seamless. I incorporated Bootstrap and designed a simple navigation by having a landing page, search page and personal collection page. I also delved into Data Modeling for this app by creating three separate collections in my Mongo Database and writing queries that pulled in data from multiple collections to one page. I used my knowledge of ERD and Class Diagrams to draw out my schemas before I began to code. In general this helped me realize how important it is to have a mapped out solution before I dive into code.

Reddit Insights

Reddit Insights with IBM Watson

Description

This app allows a user to input a Reddit Username and returns a personality profile based upon the Reddit user’s comments. Upon submission a request is made to Reddit for the comments, the comments are then sent to IBM’s Watson and run through their Personality Insights, which returns a personality profile as a JSON object. This object is parsed and displayed graphically for the end user using Chart.js.

I pair programmed the majority of this app with a peer. We both learned how to work intimately with another on a project, by respecting perspective ideas and sharing keyboard time. Building this application also provided solid experience dealing with async Javascript, making sure the data, from user input to the final personality output, flowed correctly.

Fun Finder

Fun Finder

Description

This app allows a user to input a major U.S. city and interests and returns related activities and events. On any given user submission up to 160 activities and events will be returned from requests made to the Eventful API. The user can click explore and the page quickly displays a new event. Users can also login and save events to their favorites page.

This was a group project and my focus was on the data. I wrote the code that made the API calls and organized the data with an object-oriented-programming approach. On the client side I wrote the code that randomized the events and made sure never to display the same event twice. Working in a group provided great experience dealing with git merging conflicts and delegating responsibilities appropriately.

Money and Politics

Money and Politics

Description

This app allows a user to input a zip-code and election cycle and then displays candidates, vote count, and net campaign contributions for the correlating Congressional district and election year.

This was a personal project where I first learned how to deal with async Javascript. I have data in a Mongo Database and make API calls to Sunlight Labs. The data in my database is election results parsed and inserted from excel sheets downloaded via the Federal Election Commission's website. I made calls to Sunlight Labs for candidate specific information and net contribution totals. It was a great learning experience gathering information from separate collections and making it readily available on the page for the user.
